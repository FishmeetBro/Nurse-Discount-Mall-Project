# 护士专属健康折扣商城微信小程序 Demo 全维度高标准优化 Goal

## 1. 目标名称

护士专属健康折扣商城微信小程序 Demo 全维度高标准优化（原生 WXML / WXSS / JS，纯本地演示版）

## 2. 总体目标

在不接入后端、云开发、支付、物流及第三方插件的前提下，将现有 5 页面 Demo 优化为可直接向领导演示的完整原型。合格底线为：**零报错、零错位、零违和、零卡顿**。所有数据均使用本地模拟数据与微信本地缓存，所有业务入口可响应、可回退、可重复演示。

## 3. 核心交付物

1. 五个完整页面：首页、护士认证、商品详情、合伙人中心、个人中心。
2. 四类公共组件：商品卡片、订单条目、佣金统计卡片、合规提示。
3. 统一医用蓝白设计令牌与全局样式。
4. 可直接使用的本地图片素材及标准化替换路径。
5. 本地数据、缓存封装、商品数据模块及业务交互闭环。
6. ImageGen Prompt 清单、逐页修改说明、公共组件说明、自检报告。

## 4. 六大维度验收细则

### A. 视觉美观度

| 编号 | 验收项 | 量化标准 | 验证方式 |
|---|---|---|---|
| V-01 | 设计体系统一 | 全项目只使用统一主色、辅助色、圆角、阴影、间距梯度；同类组件无不同口径 | 对照全局令牌与页面截图 |
| V-02 | 页面层级 | 每页均具备明确的主标题、核心卡片、辅助信息层级，正文/辅助文字对比度可辨 | 逐页视觉检查 |
| V-03 | 对齐与留白 | 卡片左右边距统一；同级元素基线对齐；无贴边、挤压、遮挡 | 375px 与 320px 模拟器检查 |
| V-04 | 价格表达 | 普通价与护士价层级明确，护士价醒目但不过度营销 | 首页及详情页检查 |
| V-05 | 细节一致 | 分割线、图标容器、标签、按钮、空状态风格一致 | 全页面交叉检查 |
| V-06 | 四零底线 | 零错位、零违和；无明显占位符、乱码、未完成视觉块 | 逐页截图核验 |

### B. 交互流畅度

| 编号 | 验收项 | 量化标准 | 验证方式 |
|---|---|---|---|
| I-01 | 入口可用 | 所有可点击元素均有处理函数，无点击无响应 | 自动扫描绑定事件 + 手测 |
| I-02 | 反馈及时 | 点击后 100ms 内出现视觉反馈或系统反馈；提交按钮有防重复状态 | 真机/模拟器手测 |
| I-03 | 状态过渡 | 弹层、卡片、按钮、筛选切换具备 150–300ms 柔和过渡 | 逐项手测 |
| I-04 | 防重复触发 | 审核、海报生成、加入购物车等操作具备锁或节流 | 逻辑审查与连续点击测试 |
| I-05 | 页面回退 | 五页之间跳转路径完整，无死路、无返回异常 | 完整业务路径手测 |
| I-06 | 性能底线 | 首屏无白屏；列表不执行高频 setData；无明显掉帧 | Performance 面板与代码审查 |

### C. 图片适配度

| 编号 | 验收项 | 量化标准 | 验证方式 |
|---|---|---|---|
| P-01 | 本地化 | 页面使用的正式图片全部位于项目 assets 下，不依赖网络 URL | 全项目搜索 URL |
| P-02 | 比例匹配 | Banner、商品方图、认证插图、海报、空状态均按容器比例裁切，无拉伸 | 图片尺寸与 mode 检查 |
| P-03 | 清晰度 | 源图分辨率不低于实际最大显示尺寸 2 倍 | 素材尺寸检查 |
| P-04 | 风格一致 | 统一浅蓝白、低饱和、干净医用商业插画/摄影风格 | 素材拼图对照 |
| P-05 | 可替换性 | 图片路径集中在数据模块或固定 assets 目录，替换无需改布局 | 路径与组件检查 |
| P-06 | 内容安全 | 无品牌商标、无错误医疗功效文字、无水印 | 人工审图 |

## 5. 质量红线

出现以下任一项即判定不合格：

1. 微信开发者工具出现语法错误、组件缺失、页面找不到或未处理异常。
2. 任一主页面白屏、核心入口无响应、按钮重复触发导致状态错乱。
3. 320px 小屏发生横向滚动、固定底栏遮挡内容或核心文字溢出。
4. 图片拉伸、模糊、风格冲突、含水印或含未经验证的医疗疗效文案。
5. 出现三级及以上分佣、医院内推销暗示、收益承诺或夸大疗效。
6. 引入第三方插件、远程接口、云开发、真实支付或物流调用。
7. 认证、订单或佣金关键状态在正常刷新后丢失。
8. 公共组件未实际复用，或页面仍保留对应重复结构。
