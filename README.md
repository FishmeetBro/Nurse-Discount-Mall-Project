# 护士专属健康折扣商城

面向护士群体的健康好物自用与合规分享微信小程序 Demo。项目采用微信小程序原生 WXML、WXSS 和 JavaScript 开发，所有商品、认证、订单、佣金、优惠券及支付数据均为本地模拟数据。

![护士专属健康折扣商城](assets/images/banner-nurse-health.jpg)

## 项目说明

本项目用于产品原型展示和前端交互演示，不接入真实后端、云开发、支付、物流或医疗服务，不会产生真实交易与资金结算。导入微信开发者工具后即可直接编译预览。

设计采用浅蓝与白色为主的简约医疗风，支持护士身份认证、专属价格展示、商品浏览、购物车结算、模拟支付及二级分佣数据展示。

## 功能概览

| 模块 | 主要功能 |
|---|---|
| 首页 | 医疗风 Banner、护士折扣专区、健康分类、爆款商品、普通价与护士专享价、购物车入口 |
| 护士认证 | 护士证选择、身份证输入与校验、协议确认、模拟审核、认证状态持久化 |
| 商品详情 | 商品主图、医疗器械资质说明、双价格、推广海报、原生分享、加入购物车 |
| 合伙人中心 | 专属推广链接、佣金数据、一级与二级护士统计、朋友圈素材复制 |
| 个人中心 | 认证状态、优惠券、积分展示、收藏、模拟订单、分销业绩看板 |
| 优惠券 | 可用、已使用及已过期优惠券，本地选择并参与结算金额计算 |
| 我的收藏 | 收藏商品浏览、查看详情、加入购物车及清空收藏 |
| 购物车 | 商品勾选、全选、数量修改、删除、金额合计和结算 |
| 模拟支付 | 地址与商品确认、优惠券抵扣、应付金额计算、模拟订单生成 |

## 技术特点

- 使用微信小程序原生 WXML、WXSS、JavaScript 与 JSON。
- 不使用 npm 依赖、第三方插件、云开发及外部接口。
- 使用 `rpx`、百分比和 Flex 布局适配不同尺寸设备。
- 认证、购物车、订单、优惠券及合伙人数据使用本地缓存持久化。
- 商品数据与缓存逻辑集中在 `utils/`，便于统一维护。
- 商品卡片、订单条目、佣金统计和合规提示均封装为公共组件。
- 8 张项目图片全部存放在本地 `assets/images/` 目录。
- 已锁定竖屏方向，并为按钮、卡片、标签和弹层提供轻量交互反馈。

## 运行方式

1. 下载或克隆本仓库。
2. 打开微信开发者工具。
3. 选择“导入项目”，项目目录选择仓库根目录。
4. AppID 可使用你自己的测试 AppID；仅查看界面时也可按开发者工具支持的方式使用测试号。
5. 导入后点击“编译”即可预览。
6. 如修改代码后界面未刷新，可执行“清缓存 → 全部清除”，再重新编译。

推荐演示路径：

```text
首页 → 护士认证 → 商品详情 → 生成推广海报
首页 → 购物车 → 修改数量 → 结算 → 选择优惠券 → 模拟支付
合伙人中心 → 复制推广链接 / 朋友圈文案
个人中心 → 优惠券 / 收藏 / 订单 / 分销业绩
```

## 数据与图片修改

- 商品名称、分类、价格、图片和卖点：修改 `utils/products.js`。
- 首页分类、Banner 和推荐区文字：修改 `pages/home/`。
- 合伙人、佣金、团队及素材文案：修改 `pages/partner/partner.js`。
- 个人中心订单与业绩初始数据：修改 `pages/profile/profile.js`。
- 项目图片：替换 `assets/images/` 中对应文件，并保持原文件名即可，无需调整布局。
- 本地缓存统一使用 `nurseMall:v2:` 前缀；需要重新体验初始状态时，可在微信开发者工具中清除 Storage。

## 合规声明

- 本项目仅支持一级与二级推广关系，不设置三级及以上分佣。
- 禁止在医院或其他医疗工作场所内进行推销。
- 不得利用护士身份、患者关系或工作便利开展强制推广。
- 不得夸大医疗器械疗效，不作治疗、治愈或收益承诺。
- 商品信息不构成诊疗建议，医疗器械应根据说明书规范使用。
- 所有价格、订单、佣金、物流和支付结果均为本地 Demo 数据。
