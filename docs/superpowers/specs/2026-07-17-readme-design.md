# 护士专属健康折扣商城 README 设计

## 目标

为 GitHub 仓库创建一份纯中文、项目展示型 README，使领导、产品人员和开发者能在首页快速理解项目定位、功能范围、运行方式与合规边界。

## 内容结构

1. 项目标题与一句话定位。
2. Demo 属性说明：纯前端、本地模拟数据、无真实支付及后端服务。
3. 核心功能：5 个核心页面与优惠券、收藏、购物车、模拟支付扩展闭环。
4. 技术特点：原生 WXML/WXSS/JS、公共组件、本地缓存、响应式 rpx、本地图片资源。
5. 页面与目录结构。
6. 微信开发者工具导入及运行步骤。
7. 自动检查命令与当前验证结果。
8. 数据与图片修改入口。
9. 合规声明：仅二级分佣、禁止医院内推销、不得夸大医疗器械疗效。
10. 项目用途与许可说明。

## 表达与格式

- 全文使用简体中文。
- 风格简洁、专业，不使用营销夸张措辞。
- 使用适量表格、代码块和列表提高可扫描性。
- 不写入虚假的线上地址、构建状态、下载量或真机测试结论。
- 使用仓库内 Banner 图片作为项目视觉预览，不增加外部图片依赖。

## 验收标准

- GitHub 首页可正确渲染，无断链和占位文本。
- 页面数量、目录名称、测试命令与当前源码一致。
- 明确标注所有支付、订单、佣金和物流数据均为本地模拟数据。
- README 同时存在于本地项目和远程 `main` 分支。
